AI & Fairness

How Adriano To The Star uses AI and monitors fairness

Where AI Is Used

This site uses AI in a few focused places to enhance the experience:

  • Stellar AI chat: conversational assistant for cosmic questions.
  • Smart search suggestions: AI-powered suggestions in the exoplanet database.
  • Natural-language search: "Ask in plain language" for planet discovery.
  • Image analysis: basic computer-vision helpers for astronomy-themed content.

What We Log & Why

AI interactions are logged through a dedicated AI usage logger so we can understand how features are used and improve reliability and safety over time. Typical log entries include:

  • Feature name: for example stellar-ai, planet-search, or planet-search-ai-suggestions.
  • Model identifier: which backend model or UI flow handled the request.
  • Context: high-level metadata such as success/failure and optional fairness metrics. Raw conversation content is not required for these metrics.

Fairness & Governance Signals

Agent 3 and Agent 4 introduced several governance-focused signals that are reused across features:

  • Text classification governance: user text can be classified with a source tag (for example planet-search, planet-search-nl, or stellar-ai) so that sensitive patterns can be monitored consistently.
  • AI usage + fairness metadata: many AI events attach a fairness object that summarizes simple bias metrics or model health, especially for experimental features.
  • Bias detection helpers: synthetic tests can be run through a dedicated bias-detection utility to produce small fairnessSummary objects that are logged alongside AI usage.

Developer Views & Exports

For developers and maintainers, there are several built-in tools to inspect AI behavior:

  • Analytics Dashboard cards: the Analytics Dashboard page includes AI Usage Overview and AI Fairness Overview cards that summarize recent events by feature, model, and fairness coverage.
  • My Dashboard dev buttons: logged-in users see developer buttons to export performance metrics and show AI usage / AI fairness summaries in the console.
  • Fairness exports: a dedicated fairness export helper can generate JSON snapshots of recent fairness-related events for offline analysis.

Monitoring & Safety Posture

Monitoring scripts help ensure heavy analytics are placed on the right pages and that core AI governance features are available where AI is used. Developer helpers include:

  • runMonitoringPlacementAudit() – checks which monitoring and AI scripts are loaded on the current page.
  • runRateLimitingAudit() – summarizes rate limits and feature flags for exports and RUM behavior.
  • runAIFeaturePresenceAudit() – reports which AI and governance components (usage logging, text classification, bias detection, etc.) are present.
  • logAISafetyPostureSummary() – prints a one-line summary of whether the most important AI governance features are available on the page.

For more implementation detail, see the developer documents AGENT-3-AI-GOVERNANCE-NOTES.md and AGENT-4-WRAP-UP.md in the project repository.
